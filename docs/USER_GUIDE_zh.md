# RAW 实验室指南

本指南涵盖 LumaForge 的正常工作流：

```text
单张 RAW 文件 -> 预览 -> 风格或 LUT -> 对比 -> JPEG 导出
```

你不需要在使用前理解每一个色彩管理术语。记住一条简单的规则就好：
选你想要的风格，让 LumaForge 帮你在全分辨率路径能够复现该风格之前
把住导出关。

## 1. 打开 RAW 实验室

在现代浏览器中打开 `/raw`。配备 WebGL2 的桌面浏览器是推荐的运行环境。
移动端浏览器也能处理部分文件，但较大的 RAW 文件加上浏览器内存限制，
可能让体验不够稳定。

你的照片保留在你的设备上。LumaForge 不需要账户、云端上传、原生助手、
许可证管理器或本地守护进程。

## 2. 加载一张 RAW 文件

将 RAW 文件拖入实验室或使用文件选择器。LumaForge 围绕单张照片工作流
构建，因此请从一张源文件（而非文件夹或批量）开始。

支持的文件取决于浏览器运行时能够检查和解码的内容。常见的相机 RAW
扩展名（如 `.arw`、`.nef`、`.raf`、`.rw2`、`.orf`、`.dng`、`.cr2`、
`.cr3`、`.pef` 和 `.srw`）预期可用，但并非每种相机布局都能暴露安全
导出所需的信息。

## 3. 等待可用的预览

第一张可见图像可能来自嵌入式或快速预览。当浏览器和文件允许时，
LumaForge 随后会用更高质量的预览替换它。

预览用于交互和对比。导出是独立的：最终 JPEG 在该路径就绪时通过全分辨率
Worker 路径构建。

## 4. 选择风格

最省事的做法是选一种内置风格——内置风格无需你声明 LUT 合约即可直接使用。

对于自定义 `.cube` LUT，请添加 LUT，并在 LumaForge 询问时选择输入和输出
合约。合约告诉应用该 LUT 期望什么类型的图像以及它产生什么类型的图像。

不错的起点：

- 如果 LUT 的介绍中提到 Rec.709 或 sRGB，从显示 Rec.709/sRGB
  风格合约入手。
- 如果 LUT 是为相机 Log 配置文件制作的，选择对应的相机 Log 输入系列，
  如 ARRI LogC、Sony S-Log、Fujifilm F-Log、Canon Log、Nikon N-Log、
  Panasonic V-Log 或 RED Log3G10。
- 如果不确定，请预览结果，优先选择使肤色、天空、阴影和高光看起来
  合理的合约。避免产生死黑、荧光色或褪色对比度的合约，除非那正是你
  想要的效果。

更多示例请参见 [LUT 与导出常见问题](./LUT_AND_EXPORT_FAQ_zh.md)。

## 5. 导出前对比

使用对比模式检查原始和处理后的结果。这对强烈的 LUT 尤其有用，因为
一开始感觉良好的风格可能会将肤色、天空或阴影细节推得太过。

轻度调整控件是为了帮你完成出片。它们的目的不是把应用变成完整的
桌面 RAW 编辑器。

## 6. 导出 JPEG

只有当 LumaForge 能够通过全分辨率导出路径重建所选源文件和色彩图谱时，
导出才可用。如果导出被禁用，请阅读状态消息，而不是假设预览图像可以
保存为最终结果。

这是刻意为之。LumaForge 宁可把导出拦住，也不愿悄悄导出一张低分辨率预览
或用不同于你批准的路径去生成 JPEG。

## 快速故障排除

如果预览不显示：

- 尝试使用支持 WebGL2 的现代桌面浏览器。
- 尝试较小的 RAW 文件或来自同一相机的不同文件。
- 刷新页面并重新加载文件。

如果 LUT 结果看起来不对：

- 检查 LUT 是为显示 Rec.709/sRGB 还是相机 Log 配置文件制作的。
- 如果 LUT 名称提及某个相机 Log 系列，尝试匹配它。
- 在更改多个设置之前先降低风格强度。

如果导出不可用：

- 在实验室中检查导出状态。
- 确认源文件和 LUT 合约受支持。
- 将预览视为预览，而非最终导出路径。
