# LUT 与导出常见问题

本页面解释最常困扰 RAW 摄影爱好者的问题：该选哪个 LUT 合约、为什么导出
可能不可用，以及预期的浏览器支持情况。

## 什么是 LUT 合约？

`.cube` LUT 是一张色彩查找表。它通常假设特定类型的输入图像，例如显示
Rec.709、ARRI LogC、Sony S-Log 或其他相机 Log 曲线。

合约告诉 LumaForge 两件事：

- 输入：LUT 期望什么类型的图像。
- 输出：LUT 产生什么类型的图像。

没有这些信息，同一个 LUT 可能看起来发灰、反差过大、过饱和，或完全不对劲。

## 我应该选择哪个合约？

首先参考 LUT 的名称、下载页面或文件夹名称。LUT 包通常会给出线索，
如 `Rec709`、`sRGB`、`LogC`、`SLog3`、`FLog`、`CLog`、`VLog` 或
`REDLog3G10`。

如果 LUT 面向成片视频、静态照片、Instagram 或一般照片编辑，它通常是
显示风格的 Rec.709/sRGB LUT。

如果 LUT 面向某个相机品牌或 Log 配置文件，选择对应的配置文件系列。
例如：

| LUT 线索                     | 尝试此输入系列            |
| ---------------------------- | ------------------------- |
| `Rec709`、`sRGB`、display    | Rec.709 / sRGB 风格输入   |
| `LogC`、`ARRI`               | ARRI LogC                 |
| `SLog`、`SLog3`、`Sony`      | Sony S-Log                |
| `FLog`、`F-Log`、`Fujifilm`  | Fujifilm F-Log            |
| `CLog`、`Canon Log`          | Canon Log                 |
| `N-Log`、`Nikon`             | Nikon N-Log               |
| `V-Log`、`Panasonic`         | Panasonic V-Log           |
| `REDLog3G10`、`RED`          | RED Log3G10               |

当两个选择看起来都可以时，对比肤色、天空、中性墙壁和深阴影。更好的合约
通常在添加额外校正之前看起来就已经正常。

## 为什么选错合约看起来这么糟糕？

创意 LUT 不是通用滤镜。为相机 Log 灰片设计的 LUT 与为普通显示图像
设计的 LUT，对亮度和色彩编码的假设完全不同。

应用错误的合约可能导致：

- 死黑或发灰的阴影；
- 裁切溢出的高光；
- 过饱和的红色和蓝色；
- 偏绿或偏洋红的肤色；
- 需要过多校正的平淡结果。

LumaForge 把合约选项亮出来，因为隐藏它只会让结果更不可信。

## 为什么导出被禁用？

预览和导出有不同的职责。预览应尽快可见。导出必须通过受支持的色彩路径
重建全分辨率 JPEG。

导出可能在以下情况被禁用：

- RAW 文件无法暴露安全导出所需的处理窗口信息；
- 浏览器对该文件的能力或内存不足；
- 所选 LUT 合约不完整或不受支持；
- 导出 Worker 无法证明它可以复现已预览的色彩图谱。

这是一种防护措施。LumaForge 不应仅仅为了让下载按钮出现就悄悄保存低质量
预览或更改色彩路径。

## LumaForge 会上传我的照片吗？

不会。所选 RAW 文件在浏览器中处理。正常工作流不需要账户、云端上传、
原生助手、许可证激活或本地守护进程。

## 我应该使用哪个浏览器？

使用支持 WebGL2 的现代桌面浏览器以获得最佳基线体验。较大的 RAW 文件可能
会消耗大量浏览器内存，尤其是在移动设备上。

移动端浏览器适用于实验和较小的文件，但它们不是大型全分辨率 RAW 导出的
最可靠目标。

## 这是 Lightroom 或 DaVinci 的替代品吗？

不是。LumaForge 刻意做得更聚焦。它的用途是用风格或 LUT 完成一张 RAW 照片，
并在路径安全时导出 JPEG。

当你需要目录管理、批量处理、遮罩、降噪、镜头校正、节点图或详细的专业
调色控件时，请使用完整的编辑器。

## 文件处理失败时该怎么办？

按以下顺序尝试：

1. 使用现代桌面浏览器。
2. 尝试来自同一相机的较小 RAW 文件。
3. 尝试内置风格而非自定义 LUT。
4. 如果需要自定义 LUT，从 LUT 包中确认合约。
5. 将无法支持的文件视作兼容性边界，不要强行导出。
